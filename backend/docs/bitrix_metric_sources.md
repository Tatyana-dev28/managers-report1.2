# Источники системных показателей Bitrix24

Backend не хранит данные портала и не использует базу данных. Для каждого запроса frontend передает временную авторизацию Bitrix24, выбранный период, список сотрудников и выбранные настройки источников. Backend делает REST-запросы в Bitrix24 и возвращает рассчитанные значения.

Настройки источников выбираются в интерфейсе приложения и сохраняются в настройках приложения на стороне Bitrix24 через REST, поэтому на сервере не нужно прописывать ID смарт-процессов, воронок и стадий в `.env`.

## Фильтрация

- Звонки фильтруются по `PORTAL_USER_ID`.
- Сделки, счета и смарт-процессы фильтруются по `assignedById`.
- Попадание в стадию считается через `crm.stagehistory.list`, затем найденные элементы проверяются через `crm.item.list`.
- Период считается включительно: с начала `date_from` до начала следующего дня после `date_to`.

## Показатели

| Показатель | REST-источник | Логика |
| --- | --- | --- |
| Кол-во проведенных встреч | `crm.stagehistory.list` + `crm.item.list` | Элемент выбранного смарт-процесса встреч попал в выбранную стадию проведенной встречи, ответственный сотрудник совпадает. |
| Кол-во назначенных встреч | `crm.item.list` | Элемент смарт-процесса встреч создан за период, ответственный сотрудник совпадает. |
| Сумма дозвонов и звонков | `voximplant.statistic.get` | Все звонки сотрудника за период. |
| Кол-во исходящих дозвонов | `voximplant.statistic.get` | `CALL_TYPE = 1`. |
| Кол-во исходящих успешных дозвонов | `voximplant.statistic.get` | `CALL_TYPE = 1`, `CALL_DURATION > 10`, `CALL_FAILED_CODE` пустой или `200`. |
| Кол-во входящих звонков | `voximplant.statistic.get` | `CALL_TYPE = 2` или `CALL_TYPE = 3`. |
| Кол-во отправленных КП | `crm.item.list` | Сделка создана за период в выбранной воронке холодной базы, ответственный сотрудник совпадает. |
| Кол-во отправленных договоров | `crm.stagehistory.list` + `crm.item.list` | Договор попал в выбранную стадию отправленного договора, ответственный сотрудник совпадает. |
| Кол-во подписанных договоров | `crm.stagehistory.list` + `crm.item.list` | Договор попал в выбранную стадию подписанного договора, ответственный сотрудник совпадает. |
| Кол-во отправленных счетов | `crm.stagehistory.list` + `crm.item.list` | Счет попал в выбранную стадию отправленного счета, ответственный сотрудник совпадает. |
| Кол-во оплаченных счетов | `crm.stagehistory.list` + `crm.item.list` | Счет попал в выбранную стадию оплаченного счета, ответственный сотрудник совпадает. |
| Кол-во новых сделок | `crm.item.list` | Сделка создана за период, ответственный сотрудник совпадает. |
| Кол-во успешных сделок в воронке Продажа | `crm.stagehistory.list` + `crm.item.list` | Сделка попала в выбранную успешную стадию в выбранной воронке продажи, ответственный сотрудник совпадает. |
| Сумма оплаченных счетов | `crm.stagehistory.list` + `crm.item.list` | Сумма поля `opportunity` у счетов, попавших в выбранную стадию оплаченного счета. |
